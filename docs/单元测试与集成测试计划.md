# WZMediaPlayer 单元测试与集成测试计划

## 一、单元测试

### 1.1 PlaybackStateMachine 测试

**测试文件**：`WZMediaPlay/tests/PlaybackStateMachineTest.cpp`

**测试用例**：

1. **状态转换测试**
   - 测试所有合法的状态转换
   - 测试非法的状态转换（应该失败）
   - 测试幂等性（相同状态转换应该成功）

2. **并发访问测试**
   - 多线程同时查询状态
   - 多线程同时尝试状态转换
   - 验证线程安全性

3. **状态查询测试**
   - 测试所有状态查询方法
   - 验证状态查询的准确性

**运行方式**：
```bash
# 编译测试
g++ -std=c++17 -DSTANDALONE_TEST PlaybackStateMachineTest.cpp PlaybackStateMachine.cpp -o test_state_machine

# 运行测试
./test_state_machine
```

**运行方式**：

# 编译测试
g++ -std=c++17 -DSTANDALONE_TEST ErrorRecoveryManagerTest.cpp ErrorRecoveryManager.cpp -o test_error_recovery

## 二、集成测试

### 2.1 基本播放测试

**测试场景**：

1. **正常播放测试**
   - 打开视频文件
   - 播放30秒
   - 检查画面和声音是否正常
   - 检查CPU和内存使用

2. **暂停/恢复测试**
   - 播放中暂停
   - 等待5秒
   - 恢复播放
   - 检查音视频是否同步

3. **停止测试**
   - 播放中停止
   - 检查所有线程是否正常退出
   - 检查资源是否释放

**测试步骤**：
1. 启动播放器
2. 打开测试视频文件
3. 执行测试操作
4. 检查日志和性能指标
5. 验证结果

### 2.2 Seeking测试

**测试场景**：

1. **正常Seeking测试**
   - 播放中Seek到不同位置（10%, 50%, 90%）
   - 检查画面是否正确显示
   - 检查是否花屏
   - 检查音视频是否同步

2. **快速Seeking测试**
   - 快速连续Seeking 10次
   - 检查是否崩溃
   - 检查最终位置是否正确

3. **边界Seeking测试**
   - Seek到文件开头
   - Seek到文件末尾
   - Seek到无效位置（负数、超过文件长度）

**测试步骤**：
1. 启动播放器并开始播放
2. 执行Seeking操作
3. 检查日志中的seeking相关日志
4. 验证画面和音视频同步

### 2.3 稳定性测试

**测试场景**：

1. **长时间播放测试**
   - 播放视频1小时
   - 检查内存泄漏（使用工具如Valgrind或Application Verifier）
   - 检查CPU使用（应该稳定）
   - 检查是否崩溃

2. **压力测试**
   - 快速操作（播放/暂停/Seek）
   - 检查是否崩溃
   - 检查响应时间

3. **异常文件测试**
   - 损坏的视频文件
   - 无音频的视频文件
   - 超大视频文件
   - 检查错误处理

**测试步骤**：
1. 准备测试文件
2. 执行长时间播放或压力测试
3. 监控资源使用
4. 检查日志中的错误

### 2.4 视频切换测试

**测试场景**：

1. **正常切换测试**
   - 播放视频到结束
   - 检查是否自动切换到下一个视频
   - 检查切换后播放是否正常

2. **快速切换测试**
   - 快速切换多个视频
   - 检查是否崩溃
   - 检查资源是否释放

**测试步骤**：
1. 准备播放列表
2. 播放第一个视频
3. 等待播放完成或手动切换
4. 检查切换后的播放状态

### 2.5 性能测试

**测试场景**：

1. **CPU使用率测试**
   - 播放不同码率的视频（低/中/高）
   - 记录CPU使用率
   - 检查是否在合理范围（<50% for 1080p）

2. **内存使用测试**
   - 播放视频30分钟
   - 记录内存使用
   - 检查是否有内存泄漏（内存应该稳定）

3. **渲染性能测试**
   - 记录FPS
   - 检查是否达到目标帧率（视频帧率±2）
   - 检查跳帧率（应该<5%）

**测试工具**：
- Windows Performance Monitor
- Process Explorer
- Application Verifier（内存泄漏检测）

## 三、测试自动化

### 3.1 测试脚本

**Windows批处理脚本**：
```batch
@echo off
echo Running WZMediaPlayer tests...

REM 基本播放测试
echo Testing basic playback...
WZMediaPlay.exe test_video.mp4
timeout /t 30
taskkill /F /IM WZMediaPlay.exe

REM Seeking测试
echo Testing seeking...
WZMediaPlay.exe test_video.mp4
REM 使用自动化工具发送Seek命令
timeout /t 10
taskkill /F /IM WZMediaPlay.exe

echo Tests completed!
```

### 3.2 测试数据准备

**测试视频文件**：
1. **标准测试视频**
   - 1080p, 30fps, H.264
   - 720p, 60fps, H.264
   - 4K, 30fps, H.265

2. **特殊测试视频**
   - 损坏的视频文件（部分数据损坏）
   - 无音频的视频文件
   - 超大视频文件（>10GB）

3. **3D测试视频**
   - 左右格式3D视频
   - 上下格式3D视频

## 四、测试报告

### 4.1 测试结果格式

**测试报告应包含**：
1. 测试用例名称
2. 测试结果（通过/失败）
3. 测试日志
4. 性能指标
5. 问题描述（如果失败）

### 4.2 持续集成

**建议**：
1. 每次代码提交后自动运行单元测试
2. 每日运行集成测试
3. 发布前运行完整测试套件

## 五、测试优先级

### 5.1 高优先级测试（必须通过）

1. 基本播放测试
2. Seeking测试
3. 视频切换测试
4. 稳定性测试（30分钟）

### 5.2 中优先级测试（应该通过）

1. 性能测试
2. 异常文件测试
3. 压力测试

### 5.3 低优先级测试（可选）

1. 长时间播放测试（1小时+）
2. 极端情况测试

## 六、测试执行建议

### 6.1 开发阶段

- 每次修改后运行相关单元测试
- 每次提交前运行基本集成测试

### 6.2 发布前

- 运行完整测试套件
- 进行长时间稳定性测试
- 进行性能测试

### 6.3 问题修复后

- 运行相关测试用例
- 运行回归测试
- 验证修复效果
