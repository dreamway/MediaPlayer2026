# 视频切换与崩溃修复总结

## 修复日期
2026-01-15

## 一、问题分析

### 1.1 视频切换时线程停止问题

**问题现象**：
- 日志显示：`Movie still not stopped after additional wait! Video thread may be stuck.`
- 视频播放完成后切换到下一个视频时，VideoThread无法及时停止

**根本原因**：
1. `requestStop()`只设置了`br_ = true`，但VideoThread可能在`sleep_for(10ms)`中，需要等待才能检查`br_`
2. 没有唤醒线程，让它们立即检查`br_`标志
3. 线程删除前没有确保线程完全停止

### 1.2 播放过程中崩溃问题

**问题现象**：
- 日志在2351行突然截断，说明程序崩溃
- 崩溃发生在正常播放过程中

**根本原因**：
1. VideoThread在stop()后还在调用`writeVideo()`，但OpenGL上下文可能已被销毁
2. 线程在stop()时被删除，但还在运行
3. OpenGL操作在错误的线程或无效的上下文中执行

## 二、修复内容

### 2.1 修复视频切换时线程停止问题 ✅

**修复内容**：

1. **改进PlayController::stop()**：
   - 在`requestStop()`后立即调用`emptyBufferCond_.wakeAll()`唤醒所有线程
   - 确保线程能及时检查`br_`标志并退出
   - 在删除线程前检查线程是否还在运行，避免删除正在运行的线程

2. **优化VideoThread的sleep操作**：
   - 将`sleep_for(10ms)`改为可中断的等待（循环1ms，检查`br_`标志）
   - 如果`br_`为true，立即退出循环
   - 避免在stop()时长时间阻塞

3. **优化AudioThread的sleep操作**：
   - 同样将`sleep_for(10ms)`改为可中断的等待
   - 确保AudioThread也能及时响应stop()请求

**修改文件**：
- `WZMediaPlay/PlayController.cpp`
- `WZMediaPlay/videoDecoder/VideoThread.cpp`
- `WZMediaPlay/videoDecoder/AudioThread.cpp`

### 2.2 修复播放过程中崩溃问题 ✅

**修复内容**：

1. **在VideoThread中添加br_检查**：
   - 在调用`writeVideo()`前检查`br_`标志
   - 如果`br_`为true，立即退出循环，避免继续渲染
   - 在渲染前和渲染后都检查`br_`标志

2. **在OpenGLWriter中添加安全检查**：
   - 检查`m_drawable`是否有效（避免访问已销毁的对象）
   - 检查OpenGL上下文是否有效（避免在stop()后访问已销毁的上下文）
   - 添加异常保护，避免OpenGL操作异常导致崩溃

3. **改进线程删除逻辑**：
   - 在`PlayController::open()`中，删除旧线程前确保线程已完全停止
   - 如果线程还在运行，记录错误但不删除，避免崩溃

**修改文件**：
- `WZMediaPlay/videoDecoder/VideoThread.cpp`
- `WZMediaPlay/videoDecoder/opengl/OpenGLWriter.cpp`
- `WZMediaPlay/PlayController.cpp`

## 三、关键改进点

### 3.1 线程停止机制改进

**改进前**：
```cpp
videoThread_->requestStop();
videoThread_->wait(5000);  // 可能等待5秒
```

**改进后**：
```cpp
videoThread_->requestStop();
emptyBufferCond_.wakeAll();  // 立即唤醒线程
if (!videoThread_->wait(5000)) {
    // 处理超时
}
```

### 3.2 可中断的等待

**改进前**：
```cpp
std::this_thread::sleep_for(std::chrono::milliseconds(10));
```

**改进后**：
```cpp
// 使用可中断的等待
for (int i = 0; i < 10 && !br_; ++i) {
    std::this_thread::sleep_for(std::chrono::milliseconds(1));
}
if (br_) {
    break;  // 如果收到停止信号，立即退出
}
```

### 3.3 OpenGL上下文安全检查

**新增检查**：
```cpp
// 检查OpenGL上下文是否有效
if (m_drawable->widget()) {
    QOpenGLWidget* glWidget = qobject_cast<QOpenGLWidget*>(m_drawable->widget());
    if (glWidget && (!glWidget->context() || !glWidget->context()->isValid())) {
        logger->warn("OpenGL context is invalid, skipping render");
        return;
    }
}
```

## 七、Seeking优化 (2026-01-15)

### 7.1 Seeking优化内容

**优化目标**：
- 解决Seeking多次后崩溃的问题
- 消除"先跳回原点再跳到目标点"的问题
- 优化缓冲管理，避免队列满导致的阻塞

**已完成的优化**：

1. **队列清空时机优化** ✅
   - 在`DemuxerThread::requestSeek()`时立即清空队列
   - 避免队列满导致`DemuxerThread`阻塞
   - 确保新位置的数据包能够及时放入

2. **KeyFrame直接跳转** ✅
   - 修改`Demuxer::seek()`优先使用`AVSEEK_FLAG_FRAME`
   - 直接跳到目标点附近的关键帧
   - 避免"先跳回原点再跳到目标点"的问题

3. **缓冲管理优化** ✅
   - 在seeking期间，如果队列满，直接丢弃数据包
   - 避免阻塞`DemuxerThread`
   - 非seeking期间保留超时重试机制

4. **Seeking同步机制改进** ✅
   - 优化`VideoThread`的关键帧处理逻辑
   - 在成功解码关键帧并达到目标位置后，立即清除`wasSeekingRecently_`标志
   - 确保后续帧正常渲染，避免被错误跳过

**修改文件**：
- `WZMediaPlay/videoDecoder/DemuxerThread.cpp`
- `WZMediaPlay/videoDecoder/Demuxer.cpp`
- `WZMediaPlay/PlayController.cpp`
- `WZMediaPlay/videoDecoder/VideoThread.cpp`
