### ✅ Phase 7: VideoThread Packet 处理修复（2026-01-18）

  - 负数: 错误码
- `processDecodeResult` 正确处理不同返回值
- 添加详细的调试日志

## 紧急问题：音频卡住，黑画面 🔴

**发现日期**: 2026-01-17 17:18
**问题描述**: 播放后黑画面，音频缓冲区满无法消费，视频跳帧

**修复状态**: 🔄 进行中（已优化缓冲区大小和播放启动逻辑）

### 问题分析（基于日志 L165-L187）

#### 问题1：缓冲区太小导致快速填满（L183）
```
Audio::writeAudio: Buffers full, waiting...
```

**根本原因**:
- 旧设置：9个缓冲区 x 20ms = 180ms 总缓冲
- 每次写入：38400 字节（约 20ms）
- 结果：快速填满，无法被消费

**修复**:
- 新设置：10个缓冲区 x 40ms = 400ms 总缓冲（增加 122%）

#### 问题2：OpenAL 播放未被启动（无日志）
```
❌ 日志中找不到 "PlayController::play"
❌ 日志中找不到 "Audio playback started"
```

**根本原因**:
- `PlayController::open()` 虽然转换到 Playing 状态
- 但**没有调用 `audio_->play()`** 启动 OpenAL 播放
- OpenAL 状态为 AL_INITIAL (4113) 而不是 AL_PLAYING (4114)
- `AL_BUFFERS_PROCESSED` 始终为 0

**修复**:
- 在 `PlayController::open()` 的最后调用 `audio_->play()`
- 确保文件打开后立即启动 OpenAL 播放
