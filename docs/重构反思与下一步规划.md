## 当前状态分析

### 当前架构问题

#### 1. **架构不匹配**
- **当前**：`VideoThread` → `Video::start()` → `FrameBuffer` → `currentFrame()` → `FFmpegView`
- **QMPlayer2**：`VideoThr` → `dec->decodeVideo()` → `videoWriter()->writeVideo()` → `OpenGLWriter`

**问题**：
- `Video` 类（video.cc/video.h）是旧架构的产物，包含了解码逻辑
- `VideoThread` 只是简单包装，没有实现 QMPlayer2 的架构
- `FrameBuffer` 和 `currentFrame()` 是旧架构的中间层，QMPlayer2 不需要
- `FFmpegView` 还在使用 `currentFrame()`，应该直接使用 `OpenGLWriter`

#### 2. **编译错误分析**
- `Decoder::open()` 是 protected，不能直接调用 → 应该通过 `setDec()` 设置
- `Decoder` 没有 `setTimeBase()` → 时间基准应该在解码时处理，不需要单独设置
- `Decoder` 没有 `clear()` → 应该通过 `destroyDec()` 销毁解码器
- `currentFrame()` 中有重复代码（518-620行）→ 需要清理

#### 3. **代码冗余**
- `video.cc` 中的解码逻辑应该移到 `VideoThread`
- `FrameBuffer` 的复杂同步逻辑应该简化或移除
- `currentFrame()` 方法可能不再需要（如果直接使用 OpenGLWriter）

## QMPlayer2 架构理解

### 核心流程（VideoThr::run()）
```
1. 从 PacketBuffer 获取 Packet
2. 调用 dec->decodeVideo(packet, decoded, newPixFmt, flush, hurry_up)
3. 处理 VideoFilter（如果需要）
4. 音视频同步（在 VideoThr 中处理）
5. 调用 videoWriter()->writeVideo(frame, osdList)
6. OpenGLWriter 直接渲染
```

### 关键类关系
- **AVThread**：基类，管理 `Decoder* dec` 和 `Writer* writer`
- **VideoThr**：继承 AVThread，实现视频解码和渲染循环
- **Decoder**：解码接口，由 `FFDecSW` 或 `FFDecHW` 实现
- **VideoWriter**：渲染接口，由 `OpenGLWriter` 实现
- **Frame**：统一的帧封装，隐藏硬件/软件细节

### 不需要的组件
- ❌ `FrameBuffer`：QMPlayer2 不使用帧缓冲，直接解码→渲染
- ❌ `currentFrame()`：不需要外部获取帧，直接在 VideoThr 中渲染
- ❌ `Video` 类的大部分功能：解码逻辑应该在 VideoThread 中

## 架构对比总结

### QMPlayer2 的核心架构
```
PlayClass (PlayController)
  ├── DemuxerThr (DemuxerThread)
  │   └── PacketBuffer (packet_queue)
  ├── VideoThr (VideoThread)
  │   ├── Decoder* dec (FFDecSW/FFDecHW)
  │   ├── VideoWriter* writer (OpenGLWriter)
  │   └── run() {
  │       while (!br) {
  │           packet = vPackets.fetch()
  │           dec->decodeVideo(packet, decoded, ...)
  │           // 音视频同步
  │           videoWriter()->writeVideo(decoded, osdList)
  │       }
  │   }
  └── AudioThr (AudioThread)
      ├── Decoder* dec
      └── Writer* writer
```

PlayController
