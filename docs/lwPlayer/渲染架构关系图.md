# WZMediaPlayer 渲染架构关系图

## 架构概述

WZMediaPlayer 采用双渲染架构，分别处理视频文件播放和 Camera 输入：

- **视频文件渲染**：通过 `VideoRenderer` 体系（`OpenGLRenderer` / `StereoOpenGLRenderer`）
- **Camera 渲染**：通过 `CameraOpenGLWidget` + `CameraRenderer` 体系

### 当前架构（待改进）

两者通过 `StereoVideoWidget` 统一管理，`CameraOpenGLWidget` 作为 `StereoVideoWidget` 的子组件，根据 `RenderInputSource` 切换显示。

### 改进方向（低优先级，支线任务）

**建议**：将 `CameraOpenGLWidget` 提升为与 `StereoVideoWidget` 同级的 Widget，在 `MainWindow` 中统一管理，通过 hide/show 切换。详见 `docs/lwPlayer/架构改进方案_CameraWidget同级化.md`。

**注意**：Camera 架构改进是支线任务，优先级低于核心功能修复（硬件解码、基础播放功能）。

## Renderer 切换时机

### 1. 视频文件 → Camera

**触发条件**：
- 用户点击菜单 "打开 Camera" (`on_actionOpenCamera_toggled(true)`)
- 调用 `StereoVideoWidget::SetRenderInputSource(RIS_CAMERA)`

**切换流程**：
```
MainWindow::on_actionOpenCamera_toggled(true)
  ├─> playController_->stop()  // 停止视频播放
  ├─> ui.playWidget->openCamera(cameras[0])  // 打开Camera
  ├─> ui.playWidget->startCamera()  // 启动Camera
  │     └─> SetRenderInputSource(RIS_CAMERA)  // 切换输入源
  │           ├─> videoRenderer_->widget()->hide()  // 隐藏视频Widget
  │           └─> cameraOpenGLWidget_->show()  // 显示Camera Widget
  └─> ui.playWidget->StartRendering(...)  // 启动渲染
```

### 2. Camera → 视频文件

**触发条件**：
- 用户关闭 Camera (`on_actionOpenCamera_toggled(false)`)
- 打开视频文件 (`openFile()`)

**切换流程**：
```
MainWindow::on_actionOpenCamera_toggled(false)
  ├─> ui.playWidget->stopCamera()  // 停止Camera
  ├─> ui.playWidget->closeCamera()  // 关闭Camera
  ├─> ui.playWidget->StopRendering()  // 停止渲染
  └─> (后续打开视频文件时自动切换到RIS_VIDEO_FILE)

MainWindow::openFile()
  ├─> playController_->open(path)  // 打开视频文件
  └─> (PlayController内部会设置RenderInputSource为RIS_VIDEO_FILE)
        └─> SetRenderInputSource(RIS_VIDEO_FILE)
              ├─> cameraOpenGLWidget_->hide()  // 隐藏Camera Widget
              └─> videoRenderer_->widget()->show()  // 显示视频Widget
```

### 3. 2D ↔ 3D 切换

**触发条件**：
- 用户切换 2D/3D 模式（快捷键或菜单）
- 调用 `StereoVideoWidget::SetStereoFormat()`

**切换流程**：
```
MainWindow::reply_action_3D_hotKey()
  ├─> ui.action_3D->trigger() 或 ui.action_2D->trigger()
  └─> StereoVideoWidget::SetStereoFormat()
        └─> stereoRenderer->setStereoFormat(format)
              └─> (StereoOpenGLRenderer内部切换2D/3D渲染模式)
```

**注意**：2D/3D 切换不需要切换 Widget，只需要改变渲染器的渲染模式。

## UI 组件关系

### Logo 显示 (`mWindowLogo`)

**显示时机**：
- 未播放视频时显示
- 播放视频时隐藏

**位置**：窗口中心

**管理**：`StereoVideoWidget` 内部管理

### 字幕显示 (`mSubtitleWidget`)

**显示时机**：
- 加载字幕文件后显示
- 根据播放时间自动更新

**位置**：窗口底部

**管理**：`StereoVideoWidget` 内部管理，委托给 `SubtitleWidget`

### FPS 显示 (`mFPSLabel`)

**显示时机**：
- `mShowFPS == true` 时显示
- 每秒更新一次

**位置**：右上角（10px 边距）

**管理**：`StereoVideoWidget` 内部管理

### 播放列表按钮 (`butWidget`)

**显示时机**：
- 鼠标进入窗口时显示
- 鼠标离开窗口时隐藏

**位置**：右侧中间

**管理**：`StereoVideoWidget` 内部管理，信号连接到 `MainWindow`

## CameraManager 集成计划

### 当前状态

- ✅ `CameraManager` 类已创建
- ⏸️ `MainWindow` 中尚未使用 `CameraManager`
- ⏸️ `StereoVideoWidget` 中仍保留 Camera 相关代码

### 迁移步骤

1. **在 MainWindow 中创建 CameraManager**
   ```cpp
   // MainWindow.h
   class CameraManager;
   CameraManager *cameraManager_;
   
   // MainWindow.cpp
   cameraManager_ = new CameraManager(this);
   cameraManager_->setCameraWidget(ui.playWidget->cameraOpenGLWidget());
   ```

2. **修改 MainWindow 中的 Camera 调用**
   ```cpp
   // 旧代码：
   ui.playWidget->openCamera(cameras[0]);
   ui.playWidget->startCamera();
   
   // 新代码：
   cameraManager_->openCamera(cameras[0]);
   cameraManager_->startCamera();
   ```

3. **移除 StereoVideoWidget 中的 Camera 接口**
   - 移除 `getCamerasInfo()`, `openCamera()`, `startCamera()`, `stopCamera()`, `closeCamera()`
   - 移除 `mCamera`, `mCameraSession`, `mCameraSink` 成员
   - 保留 `cameraOpenGLWidget_` 用于渲染

4. **连接 CameraManager 信号**
   ```cpp
   connect(cameraManager_, &CameraManager::cameraOpened, this, [this]() {
       ui.playWidget->SetRenderInputSource(RIS_CAMERA);
       ui.playWidget->StartRendering(...);
   });
   ```

## 数据流

### 视频文件播放数据流

```
PlayController
  └─> VideoThread (解码)
        └─> Frame
              └─> VideoRenderer::render(Frame)
                    └─> OpenGLRenderer::render()
                          └─> OpenGLCommon::updateGL()
                                └─> OpenGLWidget::paintGL()
                                      └─> 显示到屏幕
```

### Camera 播放数据流

```
QCamera
  └─> QVideoSink::videoFrameChanged
        └─> CameraManager::onCameraFrameChanged
              └─> CameraOpenGLWidget::cameraRenderer()->onCameraFrameChanged
                    └─> CameraRenderer::onCameraFrameChanged(QVideoFrame)
                          └─> CameraRenderer::renderCameraFrame()
                                └─> CameraOpenGLWidget::paintGL()
                                      └─> 显示到屏幕
```

## 关键设计决策

1. **双渲染架构**：视频文件和 Camera 使用不同的渲染体系，通过 Widget 切换实现统一显示
2. **职责分离**：`StereoVideoWidget` 负责 UI 管理和 Widget 切换，不直接处理渲染逻辑
3. **Camera 管理独立化**：`CameraManager` 独立管理 Camera 生命周期，与视频播放解耦
4. **3D 支持**：通过 `StereoOpenGLRenderer` 继承 `OpenGLRenderer`，复用 2D 渲染逻辑，扩展 3D 功能
