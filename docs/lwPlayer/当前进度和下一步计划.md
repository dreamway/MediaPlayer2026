## 当前状态

### 硬件解码修复（已完成）

**问题**：当`EnableHardwareDecoding=true`时出现传输帧错误（错误码-22）

**根本原因**：
- `hw_frame->hw_frames_ctx`为空时，`av_hwframe_transfer_data`无法工作
- 虽然初始化时设置了`codec_ctx->hw_frames_ctx`，但解码出的`hw_frame`可能没有继承这个上下文

**修复方案**：
1. ✅ 在`transferFrame`中，当`hw_frame->hw_frames_ctx`为空时，从`codec_ctx->hw_frames_ctx`获取
2. ✅ 添加详细的错误日志和回退机制
3. ✅ 确保在transfer之前正确设置`hw_frames_ctx`

**修改文件**：
- `videoDecoder/hardware_decoder.cc` - 修复transferFrame，添加hw_frames_ctx回退逻辑

**待验证**：
- ⚠️ 需要测试验证硬件解码传输是否正常工作
- ⚠️ 如果仍有问题，需要进一步调试

### NV12直接渲染优化（已完成）

**优化内容**：
- Shader已经支持NV12格式（`iVideoFormat == 1`）
- 硬件解码输出NV12格式时，如果渲染器支持NV12，直接渲染，跳过格式转换
- 优化了格式转换检查逻辑，避免不必要的NV12到YUV420P转换

**修改文件**：

**效果**：
- 减少了不必要的格式转换，提升性能

3. **色彩问题**（严重）
   - 即使更新shader到x64目录并在该目录运行，颜色仍然不对
   - 日志显示使用了external shader（ShaderManager: External shaders override internal shaders）
   - 具体日志：`e:\WZMediaPlayer_2025\x64\Release\logs\MediaPlayer_20260210003852.log#L7-7`
   - 可能原因：外部shader文件有问题，或者shader加载路径不正确，着色器编译或 uniforms 设置有问题

4. **渲染性能问题**（严重）
   - FPS显示极低（0.9）
   - 视频区域显示黑色，说明渲染失败
   - 从日志分析：视频解码成功（frame pts正确），但渲染失败
   - 可能原因：OpenGL渲染管线问题，着色器执行失败，或帧数据传输错误

5. **外部着色器覆盖问题**（严重）
   - 日志显示：`ShaderManager: External shaders override internal shaders`
   - 即使在x64目录运行，仍然使用外部shader，可能导致兼容性问题
   - 需要检查shader加载逻辑，确保使用正确的shader版本
