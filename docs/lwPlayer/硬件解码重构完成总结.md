# 硬件解码重构完成总结

## 重构目标

参考QtAV的实现，重构hardware_decoder和FFDecHW，解决硬件解码黑画面问题。

## 主要改进

### 1. get_format回调改进（参考QtAV）

**改进前**：
- 简单遍历查找硬件格式
- 没有检查硬件加速是否可用
- 没有状态跟踪

**改进后**：
- ✅ 设置`AV_HWACCEL_FLAG_ALLOW_SOFTWARE`标志（允许软件回退）
- ✅ 检查是否有硬件加速格式可用
- ✅ 跟踪硬件解码器状态（宽度、高度、profile）
- ✅ 如果尺寸和profile没变化且hwaccel_context已设置，直接复用
- ✅ 如果找不到匹配格式，回退到默认行为

**关键代码**：
```cpp
// 设置允许软件回退标志
#ifdef AV_HWACCEL_FLAG_ALLOW_SOFTWARE
    ctx->hwaccel_flags |= AV_HWACCEL_FLAG_ALLOW_SOFTWARE;
#endif

// 检查硬件加速格式可用性
bool can_hwaccel = false;
for (size_t i = 0; pix_fmts[i] != AV_PIX_FMT_NONE; i++) {
    const AVPixFmtDescriptor *dsc = av_pix_fmt_desc_get(pix_fmts[i]);
    bool hwaccel = (dsc->flags & AV_PIX_FMT_FLAG_HWACCEL) != 0;
    if (hwaccel) can_hwaccel = true;
}

// 状态跟踪和复用
if (hw_decoder->hw_width_ == hw_decoder->codedWidth(ctx) &&
    hw_decoder->hw_height_ == hw_decoder->codedHeight(ctx) &&
    hw_decoder->hw_profile_ == ctx->profile &&
    ctx->hwaccel_context) {
    return pix_fmts[i];  // 复用现有上下文
}
```

### 2. transferFrame优化

**改进前**：
- 直接设置sw_frame格式和尺寸
- 没有先unref sw_frame

**改进后**：
- ✅ 先unref sw_frame，确保它是干净的
- ✅ 从hw_frames_ctx获取sw_format
- ✅ 正确设置sw_frame的format、width、height
- ✅ 在transfer后复制属性
- ✅ 验证转换后的帧数据

**关键代码**：
```cpp
// 先unref sw_frame
av_frame_unref(sw_frame);

// 从hw_frames_ctx获取软件格式
if (hw_frame->hw_frames_ctx) {
    AVHWFramesContext *frames_ctx = (AVHWFramesContext *)hw_frame->hw_frames_ctx->data;
    if (frames_ctx) {
        sw_frame->format = frames_ctx->sw_format;
        sw_frame->width = hw_frame->width;
        sw_frame->height = hw_frame->height;
    }
}

// transfer数据
int ret = av_hwframe_transfer_data(sw_frame, hw_frame, 0);

// 复制属性
ret = av_frame_copy_props(sw_frame, hw_frame);
```

### 3. 状态跟踪（新增）

**新增成员变量**：
- `hw_width_`: 硬件解码器宽度
- `hw_height_`: 硬件解码器高度
- `hw_profile_`: 硬件解码器profile

**新增辅助方法**：
- `codedWidth()`: 获取编码宽度（参考QtAV）
- `codedHeight()`: 获取编码高度（参考QtAV）

## 注意事项

1. **Windows D3D11VA**：
   - 当前实现主要针对D3D11VA和CUDA
   - 其他平台可能需要调整

2. **软件回退**：
   - 如果硬件解码失败，会自动回退到软件解码
   - 这是QtAV的设计理念

3. **状态跟踪**：
   - 如果视频尺寸或profile变化，会重新初始化
   - 这避免了潜在的问题
