# 线程生命周期管理修复总结

## 一、问题分析

### 1.1 问题现象

从错误信息和调用堆栈分析：
- **崩溃发生在视频切换时**：播放完成后切换下一个视频时崩溃
- **崩溃位置**：`VideoThread.h:55-57` 的 `isRunning()` 方法
- **错误信息**：
  - `QMutex: destroying locked mutex`（两次）
  - `Access violation reading location 0xFFFFFFFFFFFFFFFF`
- **根本原因**：Qt自动删除对象导致悬空指针

### 1.2 根本原因

1. **Qt自动删除机制**：
   - `AVThread` 构造函数中连接了 `finished()` 信号到 `deleteLater()`
   - 当线程结束时，Qt会自动删除对象
   - 但 `PlayController` 可能还在使用 `videoThread_` 或 `audioThread_` 指针

2. **悬空指针访问**：
   - `PlayController::open()` 中检查 `videoThread_->isRunning()` 时，对象可能已被删除
   - 访问已删除的对象导致访问违例

3. **Mutex销毁警告**：
   - 对象被删除时，mutex可能还在锁定状态
   - Qt警告 "QMutex: destroying locked mutex"

## 二、修复方案

### 2.1 断开finished()信号连接

**修改文件**：
1. `WZMediaPlay/PlayController.h` - 添加 `disconnectThreadSignals()` 方法
2. `WZMediaPlay/PlayController.cpp` - 实现 `disconnectThreadSignals()` 方法
3. `WZMediaPlay/videoDecoder/VideoThread.cpp` - 析构函数中断开连接
4. `WZMediaPlay/videoDecoder/AudioThread.cpp` - 析构函数中断开连接

**修复内容**：
- 在清理线程之前，断开 `finished()` 信号连接
- 在创建新线程后，立即断开 `finished()` 信号连接
- 在析构函数中，也断开 `finished()` 信号连接

### 2.2 添加异常保护

**修改文件**：
1. `WZMediaPlay/PlayController.cpp` - 在 `isRunning()` 调用前后添加异常处理
2. `WZMediaPlay/videoDecoder/VideoThread.cpp` - 析构函数中添加异常保护
3. `WZMediaPlay/videoDecoder/AudioThread.cpp` - 析构函数中添加异常保护

**修复内容**：
- 在调用 `isRunning()` 前检查指针有效性
- 使用 `try-catch` 捕获访问违例
- 如果指针无效，设为 `nullptr` 并继续执行

### 2.3 统一线程清理逻辑

**修改文件**：`WZMediaPlay/PlayController.cpp`

**修复内容**：
- 在 `open()` 中清理旧线程时，统一使用 `disconnectThreadSignals()`
- 在 `stop()` 中清理线程时，统一使用 `disconnectThreadSignals()`
- 创建新线程后，立即断开 `finished()` 信号连接

## 四、技术细节

### 4.1 Qt的deleteLater机制

- `QObject::deleteLater()` 会在事件循环中删除对象
- 如果对象有 `finished()` 信号连接，线程结束时会自动调用
- 这可能导致悬空指针问题

### 4.2 正确的线程生命周期管理

**正确的生命周期**：
```
主线程：创建线程 -> 断开finished()连接 -> 启动线程
主线程：停止线程 -> wait() -> 检查isRunning() -> delete
主线程：析构函数 -> 断开finished()连接 -> 清理
```

**错误的生命周期（已修复）**：
```
主线程：创建线程 -> 启动线程
子线程：run()结束 -> finished()信号 -> deleteLater() ❌
主线程：检查isRunning() -> 访问已删除对象 ❌ (崩溃)
```

### 4.3 异常保护策略

1. **指针有效性检查**：在调用方法前检查指针是否为 `nullptr`
2. **异常捕获**：使用 `try-catch` 捕获访问违例
3. **安全回退**：如果指针无效，设为 `nullptr` 并继续执行

1. **线程生命周期**：必须手动管理线程的生命周期，不能依赖Qt的自动删除
2. **信号连接**：创建线程后立即断开 `finished()` 信号连接
3. **异常保护**：所有 `isRunning()` 调用都要添加异常保护
4. **指针有效性**：在使用指针前检查有效性

1. ✅ **断开finished()信号连接**：避免Qt自动删除对象
2. ✅ **添加异常保护**：防止访问已删除的对象
3. ✅ **统一清理逻辑**：使用 `disconnectThreadSignals()` 统一管理
