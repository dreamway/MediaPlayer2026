## 日志监控和异常检测（新增）

### 功能描述
在自动化测试过程中，实时读取程序运行的日志文件（`logs/MediaPlayer_{timestamp}.log`），检测异常点（warning、error、critical级别的日志），并记录到测试报告中，用于迭代优化主程序。

### 实现要点

1. **日志文件定位**：
   - 在测试开始前，记录当前日志文件路径
   - 日志文件格式：`logs/MediaPlayer_{timestamp}.log`

2. **实时日志读取**：
   - 使用Python的`tail`功能持续读取新增日志
   - 每秒检查一次新日志行
   - 解析日志级别（trace、debug、info、warn、error、critical）

3. **异常检测**：
   - 检测警告级别（warn）及以上的日志
   - 检测特定错误模式（如"CRITICAL"、"ERROR"、"Exception"）
   - 记录异常日志的时间戳、级别、消息

4. **异常记录和报告**：
   - 在测试报告中添加"异常日志摘要"章节
   - 按严重程度分类（critical、error、warning）
   - 记录异常发生的时间点和上下文

5. **配置选项**：
   ```ini
   [LogMonitoring]
   enabled=true
   level_threshold=warn  # 监测warn及以上级别的日志
   patterns=error|warning|critical|CRITICAL|ERROR|WARNING
   log_file_pattern=logs/MediaPlayer_*.log
   ```

### 实现位置
- 新建：`testing/pywinauto/log_monitor.py`
- 更新：`testing/pywinauto/test_runner.py`（集成日志监控）
- 更新：`testing/pywinauto/report_generator.py`（添加异常报告）

### 测试用例

**测试1：Seeking异常日志检测**
- 前置条件：日志监控已启用
- 操作：执行10次连续seeking测试
- 验证：
  - 如果seeking出现异常（如音频不同步），日志中应有error或warning
  - 测试报告应记录这些异常日志
  - 异常日志包含时间戳和关键信息

**测试2：播放完成异常检测**
- 前置条件：日志监控已启用
- 操作：播放视频直到完成
- 验证：
  - 如果播放完成处理出现异常，日志中应有记录
  - 测试报告应记录相关异常

**测试3：多视频切换异常检测**
- 前置条件：日志监控已启用
- 操作：连续切换5个不同的视频文件
- 验证：
  - 检测资源释放、内存泄漏相关的异常
  - 测试报告应记录相关异常

### 迭代优化流程

1. **运行测试** → 生成包含异常日志的测试报告
2. **分析异常** → 定位问题根因（如音频不同步、slider更新问题）
3. **修复问题** → 更新代码解决异常
4. **回归测试** → 重新运行测试，验证异常是否解决
5. **重复** → 持续迭代，直到测试通过且无关键异常
